# Projects

This directory holds **client project workspaces** scaffolded by the toolkit. Each
project is a self-contained delivery workspace generated by the `pp-agency` CLI and
validated against the repository checklists.

## Creating a project

```bash
# Build the CLI once
npm install && npm run build

# Scaffold a new client project here
node packages/cli/dist/index.js new-project "acme-onboarding" \
  --client "ACME Corp" \
  --type canvas-app \
  --output ./projects
```

## Expected project layout

A scaffolded project follows the structure enforced by `pp-agency validate`:

```
projects/
  <client-name>/
    project-brief.md        # Required — engagement summary, scope, stakeholders
    discovery-notes.md      # Required — discovery workshop output
    prd.md                  # Recommended — product requirements
    solution-design.md      # Recommended — technical solution design
    checklists/             # Phase checklists for this engagement
    agents/                 # Agent briefs generated for this engagement
    docs/                   # Client-facing documentation
```

## Validating a project

```bash
node packages/cli/dist/index.js validate --project ./projects/<client-name>
```

> Real client workspaces are intentionally **not** committed to this toolkit
> repository. Keep client deliverables in their own private repositories. This
> directory is tracked only so the expected layout and tooling are discoverable.
